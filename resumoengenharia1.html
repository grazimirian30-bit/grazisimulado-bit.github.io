Transformação Digital

É o processo pelo qual organizações utilizam tecnologias da informação para modificar profundamente seus processos, serviços e modelos de negócio. Não se trata apenas de informatizar tarefas, mas de promover mudanças estruturais que aumentem competitividade, agilidade e inovação.

Tecnologias impulsionadoras

  • Computação em nuvem
  • Dispositivos móveis
  • Big Data e Analytics
  • Internet das Coisas (IoT)
  • Aprendizado de máquina

Sistemas de Informação

São conjuntos de componentes interrelacionados que coletam, processam, armazenam e distribuem informações para apoiar a tomada de decisão nas organizações.

Elementos de um SI

  • Hardware
  • Software
  • Pessoas
  • Dados
  • Processos

Dado x Informação

  • Dado: fato bruto sem contexto.
  • Informação: dado tratado com significado.

Camadas da Engenharia de Software

  • Processo: estrutura do desenvolvimento.
  • Métodos: técnicas de análise, projeto e teste.
  • Ferramentas: apoio automatizado (CASE).

Principais desafios

  • Crescimento da demanda por software
  • Custos elevados
  • Requisitos mal definidos
  • Atrasos de cronograma
  • Manutenção complexa

O que é requisito

Requisito é uma condição ou capacidade que o sistema deve possuir para atender às necessidades dos usuários. A engenharia de requisitos envolve descobrir, analisar, documentar e validar essas necessidades.

Tipos de requisitos

  • Funcionais: descrevem o que o sistema faz.
  • Não funcionais: descrevem qualidade, desempenho e restrições.

Processo de Engenharia de Requisitos

  1. Concepção
  2. Levantamento
  3. Elaboração
  4. Negociação
  5. Especificação
  6. Validação
  7. Gerenciamento

Stakeholders

São todas as partes interessadas no sistema (usuários, clientes, gerentes, desenvolvedores). São a principal fonte de requisitos.

Validação de requisitos

Busca verificar validade, consistência, completude e viabilidade dos requisitos, evitando retrabalho e falhas no produto final.

Rastreabilidade

Permite manter ligações entre requisitos e avaliar o impacto de mudanças ao longo do projeto.

Mensagem central

O software tornou-se um ativo estratégico das organizações, sendo essencial para competitividade, inovação e eficiência operacional.

Pontos principais

  • Sistemas de informação apoiam decisões empresariais
  • Erros de requisitos são causa frequente de falhas
  • Software deve estar alinhado ao negócio
  • Qualidade precisa ser planejada desde o início

Papel do engenheiro de software

  • Entender o domínio do negócio
  • Comunicar-se com stakeholders
  • Modelar soluções adequadas
  • Garantir qualidade do produto
